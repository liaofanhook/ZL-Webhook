API Documentation

Complete documentation for the Quotation Webhook Handler API

Webhook Endpoint

POST /webhook/quotations

Receives quotation data via webhook. The endpoint validates the incoming data and stores it in memory.

Request Headers
Header Required Description
Content-Type Required Must be application/json
Request Body Schema
{
  "id": "string | number",           // Required: Unique quotation identifier
  "customer_name": "string",         // Required: Customer name
  "customer_email": "string",        // Optional: Customer email
  "customer_phone": "string",        // Optional: Customer phone
  "items": [                         // Required: Array of items (minimum 1)
    {
      "name": "string",              // Required: Item name
      "quantity": "number",          // Required: Item quantity
      "unit_price": "number",        // Required: Price per unit
      "description": "string"        // Optional: Item description
    }
  ],
  "total_amount": "number",          // Required: Total quotation amount
  "notes": "string",                 // Optional: Additional notes
  "valid_until": "string"            // Optional: Quotation expiry date
}
Example Request
curl -X POST {{ request.url_root }}webhook/quotations \
  -H "Content-Type: application/json" \
  -d '{
    "id": "Q-2025-001",
    "customer_name": "John Doe",
    "customer_email": "john@example.com",
    "customer_phone": "+1234567890",
    "items": [
      {
        "name": "Widget A",
        "quantity": 2,
        "unit_price": 25.50,
        "description": "High-quality widget"
      },
      {
        "name": "Service B",
        "quantity": 1,
        "unit_price": 100.00,
        "description": "Professional service"
      }
    ],
    "total_amount": 151.00,
    "notes": "Bulk discount applied",
    "valid_until": "2025-09-15"
  }'
Success Response
200 OK 
{
  "success": true,
  "message": "Quotation received successfully",
  "id": "Q-2025-001"
}
Error Responses
400 Bad Request - Invalid content type 
{
  "error": "Invalid content type",
  "message": "Content-Type must be application/json"
}
400 Bad Request - Validation failed 
{
  "error": "Validation failed",
  "message": "Missing required fields: customer_name, items"
}
500 Internal Server Error 
{
  "error": "Internal server error",
  "message": "An unexpected error occurred while processing the request"
}

API Endpoints

GET /api/quotations

Retrieves all stored quotations.

Example Response
{
  "quotations": [
    {
      "id": "Q-2025-001",
      "customer_name": "John Doe",
      "items": [...],
      "total_amount": 151.00,
      "received_at": "2025-08-15T10:30:00",
      "received_from_ip": "192.168.1.1"
    }
  ],
  "total_count": 1
}

Health check endpoint for monitoring service status.

Example Response
{
  "status": "healthy",

  "timestamp": "2025-08-15T10:30:00"
}

Testing the API

You can test the webhook endpoint using various tools:

Using curl
# Test with valid data
curl -X POST {{ request.url_root }}webhook/quotations \
  -H "Content-Type: application/json" \
  -d '{"id":"TEST-001","customer_name":"Test Customer","items":[{"name":"Test Item","quantity":1,"unit_price":50.00}],"total_amount":50.00}'

# Test with invalid data (missing required fields)
curl -X POST {{ request.url_root }}webhook/quotations \
  -H "Content-Type: application/json" \
  -d '{"id":"TEST-002"}'
Using Python requests
import requests
import json

url = "{{ request.url_root }}webhook/quotations"
data = {
    "id": "PYTHON-001",
    "customer_name": "Python Test Customer",
    "items": [
        {
            "name": "Test Product",
            "quantity": 2,
            "unit_price": 25.00
        }
    ],
    "total_amount": 50.00
}

response = requests.post(url, json=data)
print(response.status_code)
print(response.json())
Using JavaScript fetch
const data = {
    id: "JS-001",
    customer_name: "JavaScript Test Customer",
    items: [
        {
            name: "Test Service",
            quantity: 1,
            unit_price: 75.00
        }
    ],
    total_amount: 75.00
};

fetch("{{ request.url_root }}webhook/quotations", {
    method: "POST",
    headers: {
        "Content-Type": "application/json"
    },
    body: JSON.stringify(data)
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error("Error:", error));

Data Storage

Important: This application uses in-memory storage. All data will be lost when the server restarts.
Automatic Data Enhancement

When quotations are received, the system automatically adds the following metadata:

  • received_at - ISO timestamp when the quotation was received
  • received_from_ip - IP address of the sender
Duplicate Handling

If a quotation with an existing ID is received, it will update the existing record rather than creating a duplicate.

Error Handling

The API implements comprehensive error handling for common issues:

  • Invalid Content Type: Returns 400 if Content-Type is not application/json
  • Malformed JSON: Returns 400 if the request body contains invalid JSON
  • Missing Required Fields: Returns 400 with specific field names that are missing
  • Invalid Data Types: Returns 400 if fields have incorrect data types
  • Empty Items Array: Returns 400 if the items array is empty
  • Server Errors: Returns 500 for unexpected server-side errors

All error responses follow a consistent format with descriptive messages to help with debugging.
